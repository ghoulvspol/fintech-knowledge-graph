技术栈

隐私计算（Privacy-Preserving Computation）

技术原理

隐私计算是一组技术的统称，目标是让多方在不暴露原始数据的前提下完成联合计算——即"数据可用不可见"。它包含五大核心技术路线：

  1. 多方安全计算（MPC, Multi-Party Computation）：1982 年姚期智院士提出的"百万富翁问题"为开端，让多方在密码学协议下完成计算（如加和、求平均、模型训练）。常见协议：Yao 混淆电路（Garbled Circuit）、GMW 协议、秘密分享（Secret Sharing）
  2. 同态加密（HE, Homomorphic Encryption）：在密文上直接做计算，结果解密后等价于在明文上计算的结果。FHE（全同态）算力开销巨大，PHE（部分同态）已工业化
  3. 零知识证明（ZKP, Zero-Knowledge Proof）：证明者在不暴露具体信息的前提下，证明某个命题为真（如证明账户余额 > 100 万但不暴露具体金额）。zk-SNARKs、zk-STARKs 是主流变种
  4. 可信执行环境（TEE, Trusted Execution Environment）：用 CPU 硬件级安全飞地（如 Intel SGX、ARM TrustZone、AMD SEV）保证代码和数据在运行时不可被外部观察
  5. 差分隐私（DP, Differential Privacy）：通过向数据或查询结果添加精心设计的噪声，让单条记录无法被逆向识别。Apple 在 iOS、Google 在 Chrome 都有大规模工程实践

联邦学习 通常被视为隐私计算的应用范式而非独立技术，常与 MPC、DP、TEE 组合使用。

在金融场景的应用

  • 跨机构联合反欺诈：A 银行查询某客户是否在 B 银行有黑名单记录，通过 MPC 协议返回 yes/no，但不暴露黑名单全集
  • 联合信贷评分：银行 + 电商 + 通信运营商联合训练信用模型（参见 联邦学习，纵向联邦中需要 PSI 隐私集合求交对齐样本）
  • 资产证明（Proof of Reserve）：交易所用零知识证明向用户证明储备金充足，但不暴露每个客户的具体余额——是 FTX 崩塌 后行业普遍采纳的合规要求
  • 反洗钱（AML）联合模型：跨行联合识别可疑账户网络
  • 保险理赔合作：医院不交出病历，保险公司不交出理赔规则，通过隐私计算判断理赔合规
  • 数字人民币隐私设计：使用零知识证明实现"小额匿名、大额可控匿名"
  • 合规数据流通：在《数据安全法》《个人信息保护法》框架下，隐私计算是数据要素跨机构流通的关键基础设施

主要玩家与产品

中国

  • 阿里巴巴 / 蚂蚁集团摩斯（MORSE）：蚂蚁的端到端隐私计算平台，结合 MPC、TEE、联邦学习，已与多家银行联合建模；蚂蚁链 AntChain 是 ZKP 应用方
  • 京东数科 / 京东科技 PPML：京东的隐私机器学习平台，主打 TEE + 联邦学习
  • 微众银行 FATE：联邦学习框架，融合 PSI、MPC 协议（参见 联邦学习）
  • 华控清交：清华姚期智团队孵化，国内 MPC 商业化代表，服务多家股份行
  • 星云 Clustar：香港中文大学团队孵化，重点 GPU 加速同态加密
  • 矩阵元 JUZIX：早期隐私计算创业公司，被多家银行采购
  • 百融云创、同盾科技：把隐私计算作为 SaaS 输出给中小银行

海外

  • Microsoft SEAL：开源同态加密库，工业界事实标准
  • Google：Federated Analytics + 差分隐私（DP）大规模工程化先驱
  • IBM Fully Homomorphic Encryption Toolkit：FHE 商业化探索者
  • Apple：iOS 全平台差分隐私落地的最大工程实例
  • Inpher：瑞士 MPC 创业公司，服务投行
  • Duality Technologies：以色列 FHE 商业化代表
  • ZKsync / StarkWare / Aleo：以太坊生态零知识证明 L2 公司

关键挑战

  • 性能挑战：同态加密的计算开销是明文的 1000-1000000 倍，FHE 至今难以工业化大规模部署；MPC 通信开销也极高
  • 协议设计挑战：MPC 协议复杂，参与方越多协议复杂度越高，错误协议设计可能反而泄露数据
  • 可验证性挑战：监管如何审计"数据真的没出本地"？需要可信日志、第三方审计机制
  • 业务挑战：客户教育成本高，业务方难以理解"为什么你给我答案但不给我数据"；激励机制缺失
  • 标准化挑战：MPC 协议、API 规范分散，机构间互操作性差
  • 法规模糊：中国《个人信息保护法》中"数据出境"的定义是否覆盖加密后的中间计算结果？欧盟 GDPR 对"匿名化"标准要求严格

国内外发展阶段

  • 中国：场景需求最强（数据要素流通是国家战略），落地公司最多（蚂蚁、华控清交、微众等），但学术原创度仍弱于美/欧/以色列
  • 美国：学术与公链 ZKP 领先（Aleo、StarkWare），金融场景落地相对慢
  • 欧盟：因 GDPR 强制，企业有动机部署，但偏向于"数据信托/数据空间"路径而非纯 MPC
  • 以色列：MPC 与 FHE 学术与商业并重，多家创业公司（Duality、QED-it）
  • 新加坡、香港：跨境数据枢纽，是隐私计算跨境商用试验地

演变趋势

  • ZKP 大爆发：以太坊 L2 推动 ZK Rollup 工业化，金融场景下 ZKP 进入"资产证明、KYC 合规、隐私交易"主流应用
  • TEE + MPC + DP 组合方案：单一技术不足，多技术融合是趋势
  • GPU 加速的 FHE：NVIDIA、华控清交、星云 Clustar 都在研究 FHE 的 GPU 加速，可能让 FHE 在 2025-2027 年进入工业可用
  • 隐私计算芯片化：专用隐私计算芯片（如 Intel SGX2、AMD SEV-SNP、国内九霄算力卡）让性能瓶颈逐步突破
  • 数据要素市场基础设施：北京、上海、深圳数据交易所均把隐私计算作为标配
  • AI + 隐私计算融合：联邦大模型微调、隐私推理（Private Inference）成为新赛道
