核心概念

Open Banking（开放银行）

一句话定义

开放银行是指银行被法律或监管要求把客户的账户、交易、支付能力通过标准 API 开放给经过授权的第三方使用，让数据所有权回归用户、降低金融业转换成本的一整套制度安排。

起源与背景

Open Banking 的现代起点是 2016 年英国竞争与市场管理局（CMA）发布的"零售银行业市场调查最终报告"，该报告认定英国的 9 家大银行存在垄断，强制要求其于 2018 年 1 月 13 日前开放标准化 API。同年欧盟《支付服务指令第二版》（PSD2）将这一要求推广至全欧盟，并定义了两类第三方角色：AISP（账户信息服务商）和 PISP（支付发起服务商）。

更早的思想渊源可追溯到 2010 年前后欧洲的"消费者数据可携权"讨论，及美国 Plaid、Yodlee 等数据聚合商通过"屏幕抓取（screen scraping）"方式实现的非标准化"开放"。Open Banking 本质上是把这种灰色实践合法化、标准化、规模化。

监管动因有三：第一，反垄断——英国大银行的客户黏性极高（每年仅 3% 切换银行），开放数据能促进竞争；第二，普惠——让创新型金融科技公司不必申请银行牌照即可触达客户；第三，反洗钱——标准化数据流有利于监管科技应用。

核心要义

第一，开放银行的核心是数据可携权，不是技术问题。 技术上 API 开放并不困难，难点在于权属界定：账户余额、交易明细这些数据是属于银行还是属于用户？英国 FCA Sandbox 与欧盟PSD2都明确：数据属于用户，银行是受托保管人，用户授权后必须提供。

第二，开放银行有两种形态——数据开放和能力开放。 数据开放（AISP）让第三方能读取账户信息，催生了财富管理聚合（Yolt）、信用评分（Credit Kudos）等产品；能力开放（PISP）让第三方能直接发起转账，绕过卡组织，是对 Visa/Mastercard 双寡头的实质冲击，2024 年英国通过 PISP 的支付笔数同比增长 73%。

第三，开放银行不是技术标准问题，而是商业模式问题。 API 接入虽是免费的，但银行通过"高级 API"（信用数据、账户开立）反向变现，形成"基础免费、增值收费"的二级市场。德国 Deutsche Bank、西班牙 BBVA 都设有专门的 API 商业化部门。

在中国

  • 主要玩家：微众银行（最早开放的银行）、网商银行、招商银行、平安银行、浦发银行、工商银行
  • 典型产品：浦发银行 API Bank（2018）、招行"招赢通"、工行"工银 API"
  • 监管态度：中国并无强制开放银行立法。央行《商业银行应用程序接口安全管理规范》（JR/T 0185-2020）属推荐性技术标准；2021 年《征信业务管理办法》和 2022 年《数据出境安全评估办法》对数据流转构成结构性约束
  • 与国外的差异：中国走的是"市场驱动+技术标准引导"的渐进路径，没有像欧盟那样的强制时间表；微信、支付宝实际上承担了部分类似 PISP 的功能，但属于支付机构而非银行

在海外

  • 主要玩家：英国 OBIE 标准下 9 大银行 + 数百家 TPP（第三方服务商）；欧盟 PSD2 框架下数千家 AISP/PISP；美国走"市场驱动 + CFPB 1033"路径，主要聚合商是 Plaid、Finicity、MX、Yodlee
  • 典型产品：英国 Yapily、TrueLayer（PISP）、Tink（被 Visa 2022 年收购）、美国 Plaid
  • 主要监管框架：英国 OBIE 标准（2018）、欧盟 PSD2（2018 生效）+ PSD3（2026 预计生效）、美国 CFPB 1033 条款（2024 年 10 月最终化）、澳大利亚 CDR、巴西 Open Finance（2021）
  • 商业模式特点：聚合商（Plaid）按 API 调用次数或月活收费；TPP 多为 SaaS 模式

商业模式

  • API 用量计费：聚合商对 TPP 收费，Plaid 对每客户每月约 0.5-2 美元
  • PISP 支付费：单笔 0.1-0.3 英镑，远低于卡组织 1.5%
  • 数据增值：信用评分、财富聚合、KYC 数据
  • 银行反向变现：高级 API（Premium API）按调用收费

风险与争议

  • 责任分担：欺诈发生时是银行、TPP 还是用户负责？2022 年英国 PSR 强制要求银行对授权欺诈"自愿赔付"
  • 数据安全：API 接入面扩大意味着攻击面扩大，2023 年欧盟 EBA 报告指出 12% 的 TPP 存在严重安全漏洞
  • 大科技进入：Plaid、Apple、Google 通过开放银行获取金融行为数据，CFPB 2024 年规则明确禁止聚合商对二级数据的二次销售
  • 银行哑管道化：开放银行让银行从"客户拥有者"沦为"基础设施提供商"，部分中小银行抗拒落地

相关概念

  • 嵌入式金融——开放银行是其技术基础
  • PSD2——开放银行的欧洲法律基础
  • Plaid——美国开放银行事实标准
  • CBDC——可能成为下一代开放金融的载体
  • RegTech——开放银行带动了合规科技

典型案例

  • Plaid：通过聚合美国超 12000 家银行 API，成为美国 Open Banking 事实标准，2020 年 Visa 宣布 53 亿美元收购被 DOJ 反垄断阻止
  • 微众银行：中国最早实践开放银行的民营银行，"Welab Bank"模式输出至香港
  • TrueLayer：英国领先 PISP，2024 年完成 1.3 亿美元 D 轮
